# Release Notes

## Release 1.5

**Developer Flow**

---

## Overview

Release 1.5 focuses on **Developer Flow**.

Rather than expanding AI capabilities, this release improves the day-to-day experience of developing software with Neovim-AIDE. The goal is to reduce friction, minimise context switching and help developers remain focused while working within Neovim.

The release introduces richer code-aware workflows, improved navigation, expanded explain and review capabilities, and a comprehensive refresh of the project's documentation.

As with previous releases, Neovim-AIDE continues to favour explicit developer control, runtime validation and observable workflows over autonomous automation.

---

# Highlights

## Explain Current Buffer

A new **Explain Current Buffer** workflow explains the complete contents of the active buffer using the existing AI execution pipeline.

The workflow automatically adapts to the current buffer type, allowing both source code and documentation to be explained appropriately.

---

## Expanded Code-Aware Workflows

Release 1.5 extends AI-assisted workflows beyond individual lines and visual selections.

New workflows include:

- Explain Current Function
- Explain Current Class
- Review Current Function
- Review Current Class

These additions make it easier to understand unfamiliar code without leaving the editor.

---

## Improved Developer Navigation

Developer Flow has been improved through additional navigation and editing workflows that reduce context switching while working in larger codebases.

Navigation improvements integrate naturally with existing Neovim editing workflows and remain fully discoverable through which-key.

---

## Documentation Refresh

The project documentation has been comprehensively refreshed.

Updated documentation includes:

- README
- Installation Guide
- Documentation Index
- Command Reference
- Architecture Guide
- Contributor Guide

The documentation now reflects the mature R1.5 implementation while providing a more consistent product identity, terminology and engineering philosophy.

---

# Engineering Improvements

Release 1.5 continues the project's emphasis on engineering quality.

Highlights include:

Release 1.5 remains fully compatible with existing Neovim-AIDE workflows and configuration.
