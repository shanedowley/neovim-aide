# Neovim-AIDE Agent Instructions

Neovim-AIDE is a Neovim IDE built around the principle:

> Human-controlled. AI-assisted.

Before performing substantial analysis, planning or implementation work in this repository, read the Neovim-AIDE AI Context Pack.

Read the documents in this order:

1. `docs/ai/README.md`
2. `docs/ai/AI_DEVELOPMENT_MODEL.md`
3. `docs/ai/PROJECT_CONTEXT.md`
4. `docs/ai/PRODUCT_PHILOSOPHY.md`
5. `docs/ai/DEVELOPMENT_GUIDE.md`
6. `docs/ai/AI_WORKING_AGREEMENT.md`
7. `docs/ai/ROADMAP.md`
8. `docs/ai/PROJECT_TERMINOLOGY.md`

These documents describe the project's development model, intent, philosophy, development practices, strategic direction and preferred terminology.

They complement—but never replace—the source code and technical documentation.

If a conflict exists between these documents and the implementation, the implementation is authoritative.

When contributing:

- understand the project before implementing
- seek understanding before implementation
- preserve existing architecture
- prefer small, focused and incremental changes
- avoid speculative refactoring
- validate meaningful changes
- update relevant documentation
- explain risks and trade-offs clearly
- remember that the AI assists and the developer decides

When collaborating:

- ChatGPT is the preferred partner for architecture, planning, design, documentation and technical discussion.
- Codex is the preferred partner for implementation, refactoring, debugging and repository changes.
- The developer owns all technical and product decisions.
- AI assistants should complement one another rather than duplicate responsibilities.

The objective is not to maximise AI involvement.

The objective is to maximise software quality.
